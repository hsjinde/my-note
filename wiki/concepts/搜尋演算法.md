# 搜尋演算法

從資料中找出目標值的方法。效率取決於資料是否已排序。

## 搜尋法比較

| 演算法 | 前提 | 時間複雜度 | 空間 |
|---|---|---|---|
| Linear Search | 無 | O(n) | O(1) |
| Binary Search | 已排序 | O(log n) | O(1) |
| Interpolation Search | 已排序 + 均勻分布 | O(log log n) avg | O(1) |

## Linear Search（線性搜尋）
從頭逐一比對，不需排序。簡單但 O(n)。

## Binary Search（二分搜尋）
取中間值比較 → 目標小則搜左半、大則搜右半 → 每回合範圍減半。
- **前提**：資料必須已排序
- 邊界條件是常見 bug 來源（`start <= end` vs `start < end`）

## Interpolation Search（內插搜尋）
Binary Search 改良版。用斜率公式估算目標位置：
`mid = start + (target - arr[start]) * (end - start) / (arr[end] - arr[start])`
- 資料均勻分布時 O(log log n)
- 分布不均時可能退化

## DFS 與 BFS

用於走訪樹節點或圖頂點。

### DFS（深度優先搜尋）
- 用 **Stack**（或遞迴）
- 沿一條路走到底 → 回退 → 換條路
- 應用：路徑搜尋、連通分量、拓撲排序

### BFS（廣度優先搜尋）
- 用 **Queue**
- 逐層走訪所有相鄰節點
- 應用：最短路徑（無權圖）、層序走訪

## 費波那契優化

遞迴版費波那契 O(2^n)，大量重複計算。三種優化：

| 方法 | 時間 | 空間 | 策略 |
|---|---|---|---|
| 遞迴 | O(2^n) | O(n) | 原始 |
| DP 陣列 | O(n) | O(n) | 記憶化 |
| 變數滾動 | O(n) | O(1) | 只存前兩項 |

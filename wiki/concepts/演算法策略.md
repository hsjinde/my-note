# 演算法策略

解決演算法問題的五大常見策略。

## Divide and Conquer（分治法）

將大問題拆成**獨立**子問題 → 各自解決 → 合併結果。

應用：
- [[排序演算法#Merge Sort]]、[[排序演算法#Quick Sort]]
- [[搜尋演算法#Binary Search]]

## Dynamic Programming（動態規劃）

類似分治，但子問題**會重複**。用記憶化（Memoization）或表格儲存子問題解，避免重複計算。**空間換時間**。

應用：
- [[搜尋演算法#費波那契優化]]
- LeetCode：Climbing Stairs、Coin Change、Edit Distance、LCS

## Greedy Algorithm（貪婪法）

每步選**當下最優**，期望全局最優。不保證最佳解，但對特定問題有效。

應用：
- 找零錢（硬幣數量最少）
- Activity Selection、Huffman Coding
- Dijkstra 最短路徑

## Backtracking（回溯法）

列出所有分支 → 排除不可能的 → 嘗試一個分支 → 失敗則退回上一層換分支。避免無效搜索。

應用：
- N-Queens、Sudoku Solver
- DFS 走訪

## Branch and Bound（分支界定法）

類似地毯式搜索，但用上下界剪枝。列出所有分支 → 用界限函式排除不可能最佳解的分支。

應用：
- 旅行商問題（TSP）
- BFS 搭配剪枝

## 策略選擇指南

| 問題特徵 | 建議策略 |
|---|---|
| 可拆成獨立子問題 | 分治法 |
| 子問題重疊 | 動態規劃 |
| 每步可貪心 | 貪婪法 |
| 需窮舉但可剪枝 | 回溯法 |
| 最佳化 + 可界定 | 分支界定法 |

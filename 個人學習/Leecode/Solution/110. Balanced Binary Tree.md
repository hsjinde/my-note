## 思路總覽 (Overview & Interview Plan)

### 中文解題計畫
這題的核心在於判斷二元樹中「每個節點」的左右子樹高度差是否不超過 1。
1. **演算法選擇**：採用 **後序遍歷 (Post-order Traversal)** 的自底向上 (Bottom-up) 遞迴。
2. **具體步驟**：
   - 遞迴檢查左子樹高度。
   - 遞迴檢查右子樹高度。
   - **剪枝 (Pruning)**：若子樹已經不平衡（回傳 -1），則立即向上層回傳 -1，不再進行多餘運算。
   - **計算高度**：若目前節點平衡，回傳當前子樹的真實高度 `max(left, right) + 1`。

### English Interview Plan
The objective is to determine if a binary tree is height-balanced. A height-balanced tree is defined as a binary tree in which the left and right subtrees of every node differ in height by no more than 1.
1. **Strategy**: Utilize a **Bottom-up Recursion** approach to achieve $O(N)$ efficiency.
2. **Execution**:
   - Perform a **Post-order Traversal** to visit child nodes before the parent.
   - **Base Case**: A null node has a height of 0.
   - **Recursive Step**: Compute the height of left and right subtrees. If any subtree returns -1 (indicating it's unbalanced), propagate -1 upwards.
   - **Balance Check**: If the absolute difference $|left - right| > 1$, return -1. Otherwise, return the actual height of the current node.

## 演算法詳解

本演算法避開了傳統「Top-down」方法中重複計算高度的問題（Top-down 會導致 $O(N^2)$）。



### 遞迴狀態變化 (State Transitions)
以一個簡單節點為例：
1. `checkDepth(None)` -> 回傳 `0`。
2. `left = checkDepth(node.left)`
3. `right = checkDepth(node.right)`
4. 判斷條件：
   - 如果 `left == -1` 或 `right == -1`：說明下方已有不平衡，維持 `-1`。
   - 如果 `abs(left - right) > 1`：目前節點失衡，標記為 `-1`。
   - 否則：回傳當前最大深度 `max(left, right) + 1`。

## 程式碼實作

以下為 Python 的高效實作版本：
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def isBalanced(self, root: Optional[TreeNode]) -> bool:
        
        # 定義輔助函式：回傳高度，若不平衡則回傳 -1
        def checkDepth(node):
            # Base Case: 空節點高度為 0
            if not node:
                return 0
            
            # 遞迴檢查左子樹 (Bottom-up)
            left = checkDepth(node.left)
            if left == -1: 
                return -1  # 提早剪枝
            
            # 遞迴檢查右子樹
            right = checkDepth(node.right)
            if right == -1: 
                return -1  # 提早剪枝

            # 核心邏輯：檢查當前節點是否平衡
            if abs(left - right) > 1:
                return -1
            
            # 若平衡，則回傳該節點的高度
            return max(left, right) + 1

        # 若最終結果不是 -1，代表全樹平衡
        return checkDepth(root) != -1
```

## 正確性說明

1. **不重複 (No Redundancy)**：每個節點僅被訪問一次（自底向上），高度資訊隨遞迴回傳，不需要在每個節點重新呼叫 `getHeight` 函式。
2. **不遺漏 (Completeness)**：後序遍歷確保了在判斷父節點前，所有的子節點狀態都已被驗證。
3. **邊界條件**：
   - `root = None`：回傳 `True`（空樹符合定義）。
   - 只有一個節點：回傳 `True`。
   - 極度傾斜的樹（Skewed Tree）：遞迴會一路檢查到葉子節點並正確回傳高度差。

## 複雜度分析

- **Time Complexity**: $O(N)$
  - 其中 $N$ 為節點總數。因為我們使用後序遍歷，每個節點只會被訪問一次，所有的判斷（計算絕對值與最大值）均為 $O(1)$ 常數操作。
- **Space Complexity**: $O(H)$
  - 其中 $H$ 為樹的高度。主要開銷來自於遞迴調用棧 (Recursion Stack)。在最壞情況下（如鏈狀樹），空間複雜度為 $O(N)$；在最佳情況（完全平衡樹）下，空間複雜度為 $O(\log N)$。

## 常見誤區

1. **Top-down 陷阱**：初學者常寫出在 `isBalanced` 裡呼叫 `self.getHeight(root.left)` 與 `self.getHeight(root.right)`。這會導致大量的重複計算，時間複雜度退化至 $O(N^2)$。
2. **忽略剪枝**：在遞迴中，如果發現 `left == -1` 卻沒有立即 return，而是繼續跑完 `right = checkDepth(node.right)`，雖然結果正確，但在面試中會被視為對遞迴效率理解不足。
3. **定義混淆**：誤以為只要「總左高度」等於「總右高度」就是平衡。實際上，定義是「**每一個節點**」的左右子樹都要符合條件。

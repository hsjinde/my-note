# 資料結構總覽

資料結構是電腦儲存與組織資料的方式，分為**線性**與**非線性**兩大類。搭配 [[排序演算法]] 與 [[搜尋演算法]] 解決各類問題。

## 複雜度速查

| 結構 | 存取 | 搜尋 | 插入 | 刪除 |
|---|---|---|---|---|
| Array | O(1) | O(n) | O(n) | O(n) |
| Linked List | O(n) | O(n) | O(1) | O(1) |
| Stack | — | O(n) | O(1) | O(1) |
| Queue | — | O(n) | O(1) | O(1) |
| Hash Table | — | O(1) avg | O(1) avg | O(1) avg |
| BST | — | O(log n) avg | O(log n) avg | O(log n) avg |
| Heap | — | O(n) | O(log n) | O(log n) |

## 線性結構

### Array（陣列）
- 連續記憶體，固定大小
- 索引存取 O(1)，插入/刪除需搬移
- 多维陣列：二維、三維

### Linked List（鏈結串列）
- 非連續記憶體，節點由指標串接
- **單向**：Head → ... → Null
- **雙向**：前後指標，可雙向走訪
- **環狀**：尾部指回頭部
- 插入/刪除 O(1)（已知位置），搜尋 O(n)

### Stack（堆疊）
- **LIFO**（後進先出）
- 操作：push / pop / peek
- 應用：瀏覽器上一頁、復原、函式呼叫堆疊

### Queue（佇列）
- **FIFO**（先進先出）
- 操作：enqueue / dequeue / peek
- 應用：列印佇列、BFS

### Hash Table（雜湊表）
- 鍵值對應，雜湊函式決定存放位置
- **碰撞處理**：線性探測、平方探測、鏈結法（Chaining）
- **雜湊函式**：除法、中間平方法、折疊相加法、數位分析法
- 應用：加密、解密、快取

## 非線性結構

### Tree（樹）
- 階層結構：Root → Parent → Children → Leaf
- n 個節點 = n-1 條邊
- 術語：Level、Degree、Depth/Height

### Binary Tree（二元樹）
- 每節點最多 2 子節點
- **完全二元樹**：除最後一層外皆滿
- **滿二元樹**：每層皆滿
- **歪斜樹**：全左或全右
- 表示法：陣列（索引公式）、鏈結串列

### Binary Tree Traversal（走訪）
- **DFS**：前序 NLR、中序 LNR、後序 LRN
- **BFS**：層序走訪（Queue）

### Binary Search Tree（BST）
- 左 < Root < 右，無重複值
- 搜尋/插入/刪除 O(log n) avg
- 刪除：葉直接刪、單邊取代、雙邊取左最大或右最小

### Heap（堆積）
- **Max-Heap**：父 ≥ 子，Root 為最大值
- **Min-Heap**：父 ≤ 子，Root 為最小值
- 應用：優先佇列、[[排序演算法#Heap Sort]]
- 操作：push → heapify up、pop → heapify down

### Graph（圖）
- G = (V, E)，頂點 + 邊
- **有向圖** vs **無向圖** vs **加權圖**
- 表示法：相鄰矩陣 O(V²)、相鄰串列 O(V+E)
- 術語：Path、Cycle、Degree（入/出分支度）、Complete Graph
- 走訪：[[搜尋演算法#DFS 與 BFS]]

### Recursion（遞迴）
- 將大問題拆成小問題，自己呼叫自己
- 條件：遞迴呼叫 + 終止條件
- 費波那契數列是經典案例（見 [[搜尋演算法#費波那契優化]]）

# 🛠️ Claude Code 系統可用技能整理

本文件整理了目前在 **Claudian** 系統中可供呼叫的各項專屬技能 (Skills)。這些技能能幫助我們更高效地處理 Obsidian 筆記、進行代碼開發、繪製圖表或執行特定任務。

---

## 📝 Obsidian 專屬技能

> [!tip] 筆記與 Canvas 管理
> 這些技能是 Claudian 作為 Obsidian 原生 AI 的核心能力，可直接操作您的 Vault 內容。

### 1. `obsidian-markdown`
*   **用途**：建立與編輯符合 **Obsidian Flavored Markdown** 規範的筆記。
*   **支援語法**：
    *   雙向連結：`[[筆記名稱]]`
    *   內容嵌入：`![[嵌入內容]]`
    *   提示方塊 (Callouts)：`> [!type]`
    *   屬性/前言 (YAML Frontmatter)
    *   標籤與註解：`#tag`, `%%註解%%`
*   **何時使用**：撰寫新筆記、重構筆記格式、添加屬性或建立豐富的樣式時。

### 2. `json-canvas`
*   **用途**：建立與編輯 JSON Canvas 檔案 (`.canvas`)。
*   **支援功能**：操作節點 (nodes)、連結線 (edges)、群組 (groups) 及其連接關係。
*   **何時使用**：處理 `.canvas` 檔案、建立視覺化畫布、心智圖、流程圖等。

### 3. `obsidian-canvas-creator`
*   **用途**：從文本內容建立 Obsidian Canvas 檔案。
*   **支援功能**：支援心智圖 (MindMap) 與自由排版 (freeform) 佈局。
*   **何時使用**：當需要將文本內容轉化為互動式畫布、心智圖，或以空間形式組織資訊時。

### 4. `obsidian-bases`
*   **用途**：建立與編輯 Obsidian Bases 檔案 (`.base`)。
*   **支援功能**：設定資料庫視圖 (views)、篩選器 (filters)、公式 (formulas) 與摘要。
*   **何時使用**：處理表格視圖、卡片視圖，或需要建立筆記的資料庫管理系統時。

### 5. `obsidian-cli`
*   **用途**：透過 Obsidian 命令行工具 (CLI) 與 Vault 進行深度互動。
*   **支援功能**：讀取、建立、搜尋、管理筆記與任務屬性，以及外掛 (plugin)/主題開發調試（如重新載入外掛、執行 JS、捕捉錯誤、截圖及 DOM 檢查）。
*   **何時使用**：當需要進行 Vault 全域搜尋、批次管理筆記或調試外掛時。

## 📊 圖表與視覺化技能

### 6. `excalidraw-diagram`
*   **用途**：從文本內容生成 Excalidraw 圖表。
*   **支援功能**：支援三種輸出模式：Obsidian (`.md`)、標準 (`.excalidraw`) 與動畫 (`.excalidraw` 包含動畫順序)。
*   **何時使用**：當需要畫圖、流程圖、思維導圖、視覺化、標準 Excalidraw 或動畫圖時。

### 7. `mermaid-visualizer`
*   **用途**：將文本內容轉換為專業的 Mermaid 圖表，適用於簡報與文件。
*   **支援功能**：流程圖、系統架構圖、比較圖、心智圖等，並具備內建語法錯誤預防機制。
*   **何時使用**：當需要視覺化概念、建立流程圖或將文本製作成圖表時。

## 🌐 網頁與資料庫處理技能

### 8. `defuddle`
*   **用途**：網頁 Markdown 提取工具。
*   **功能**：從網頁 URL 提取乾淨的 Markdown 格式內容，自動去除無效的雜訊（如導覽列、側邊欄、頁尾等），以節省 Token。
*   **何時使用**：**推薦** 用於閱讀/分析線上文件、部落格文章、新聞等標準網頁（如果網頁結尾是 `.md`，則直接使用標準 WebFetch 即可）。

## 🛠️ 開發與系統調試技能

### 9. `claude-api`
*   **用途**：建置、調試與優化 Claude API 或 Anthropic SDK 應用程式。
*   **功能**：處理 Prompt 快取 (Prompt Caching)、API 版本遷移（例如 4.5 升級至 4.7 等）及相關核心功能優化。
*   **何時使用**：當代碼中引入 `anthropic` 或 `@anthropic-ai/sdk` 時。

### 10. `run`
*   **用途**：啟動並執行當前專案的應用程式以確認修改結果。
*   **何時使用**：當需要確認本地修改在真實 App 中是否運作正常時。

### 11. `verify`
*   **用途**：執行測試或手動驗證代碼變更是否符合預期。
*   **何時使用**：在提交 Pull Request 前，或要手動確認某個 Bug 已被成功修復時。

### 12. `code-review`
*   **用途**：審查修改後的代碼品質、重用性與效率，並直接修復發現的問題。

### 13. `security-review`
*   **用途**：對當前分支上的暫掛變更進行安全性審查。

### 14. `review`
*   **用途**：審查 GitHub Pull Request。

### 15. `init`
*   **用途**：在專案根目錄初始化一個全新且標準的 `CLAUDE.md` 說明文件。

---

## ⚙️ 系統設定與自動化技能

### 16. `update-config`
*   **用途**：透過 `settings.json` 設定 Claude Code 的運行環境與 Hooks。
*   **功能**：設定自動化行為（例如「每次...時執行...」）、授權權限（如允許 npm 指令）、環境變數等。

### 17. `keybindings-help`
*   **用途**：自訂與管理鍵盤快捷鍵。
*   **功能**：協助修改 `~/.claude/keybindings.json`，綁定快捷鍵。

### 18. `fewer-permission-prompts`
*   **用途**：掃描對話歷史，優先將常用的唯讀指令加入 `settings.json` 允許清單中，以減少權限詢問視窗。

### 19. `loop`
*   **用途**：在指定的時間間隔內循環執行某個 Prompt 或斜線指令（例如每 5 分鐘檢查一次部署狀態），不帶間隔參數則由模型自定節奏。

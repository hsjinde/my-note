## 題目描述 (中文摘要)
給定兩個**非空**的鏈結串列 (Linked List)，代表兩個非負整數。這些數字是以**逆序 (Reverse Order)** 儲存的，也就是說個位數位於鏈結串列的開頭。請將這兩個數字相加，並以相同的形式回傳一個表示和的鏈結串列。

**範例：**
輸入：`l1 = [2,4,3]`, `l2 = [5,6,4]`
輸出：`[7,0,8]`
解釋：342 + 465 = 807

## 思路總覽 (Overview & Interview Plan)

這道題的核心在於模擬小學數學的「直式加法」，但因為鏈結串列已經是逆序存儲（個位在頭），這剛好符合我們從低位算到高位的習慣，大大簡化了難度。

### 中文解題計畫
在面試中，我會建議採取以下步驟：
1.  **哨兵節點 (Dummy Node)**：建立一個虛擬頭節點 `dummy`，它的 `next` 指向結果鏈結串列的真正的頭。這能避免處理「頭節點是否為空」的邊界判斷。
2.  **指針操作 (Pointer Manipulation)**：使用三個變數：`cur` (指向結果鏈結串列的當前節點)、`carry` (進位值)、以及原有的 `l1` 與 `l2`。
3.  **統一迴圈邏輯**：使用 `while (l1 或 l2 或 carry)` 作為循環條件。這能優雅地處理兩個鏈結串列長度不同，或者最後還有進位 (如 5+5=10) 的情況。
4.  **數值計算**：
    - 若鏈結串列已走完，則該位數視為 0。
    - 計算 `sum = val1 + val2 + carry`。
    - 更新 `carry = sum // 10`，新節點數值為 `sum % 10`。

### English Interview Plan
To solve this problem, I will simulate the elementary math addition process digit by digit.

1.  **Initialize a Dummy Head**: Create a `dummy` node to simplify the handling of the result list's head pointer.
2.  **Iterate with a Loop**: Use a `while` loop that continues as long as `l1`, `l2`, or a non-zero `carry` exists. This handles lists of unequal lengths and final carry-overs gracefully.
3.  **Digit Extraction & Addition**:
    - Extract values from `l1` and `l2` (treating `null` nodes as 0).
    - Calculate the `current_sum` including the `carry`.
    - Update the `carry` for the next position using integer division (`//`).
    - Create a new node with the remainder (`%`) and attach it to the result list.
4.  **Pointer Movement**: Advance pointers (`l1`, `l2`, and `current`) accordingly.

## 演算法詳解

讓我們拆解迴圈內的每一次迭代：

1.  **對齊與取值**：
    我們同時遍歷 `l1` 和 `l2`。因為長度可能不同（例如 `99 + 1`），如果某個鏈結串列已經指到了 `None`，我們就將其值視為 `0`。
    $$x = l1.val \text{ if } l1 \text{ else } 0$$
    $$y = l2.val \text{ if } l2 \text{ else } 0$$

2.  **計算總和與進位**：
    將兩個數字與上一輪的進位 `carry` 相加。
    $$total = x + y + carry$$
    新的進位：$$carry = \lfloor total / 10 \rfloor$$
    當前位數的值：$$digit = total \pmod{10}$$

3.  **連接節點**：
    創建一個值為 `digit` 的新節點 `ListNode(digit)`，並將其接在 `cur.next` 後面，然後移動 `cur` 指針。

4.  **推進輸入指針**：
    如果 `l1` 或 `l2` 還沒走完，將它們移向 `next`。

## 程式碼實作

```python
from typing import Optional

# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def addTwoNumbers(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
        # 使用 Dummy Node 簡化頭部操作
        dummy = ListNode(0)
        cur = dummy
        carry = 0
        
        # 只要 l1, l2 還有節點，或是還有進位 (carry > 0)，都需要繼續運算
        while l1 or l2 or carry:
            # 取值：若節點存在取其值，否則視為 0
            x = l1.val if l1 else 0
            y = l2.val if l2 else 0
            
            # 計算當前總和 (含進位)
            xy_sum = x + y + carry
            
            # 更新進位與當前位數的值
            carry = xy_sum // 10
            new_val = xy_sum % 10
            
            # 建立新節點並連接
            cur.next = ListNode(new_val)
            cur = cur.next
            
            # 移動指針
            if l1: l1 = l1.next
            if l2: l2 = l2.next
            
        return dummy.next
```
## 正確性說明
1. **不遺漏 (Handling Different Lengths)**：
   透過 `x = l1.val if l1 else 0` 的寫法，即便其中一個鏈結串列較短，演算法也會自動補 0 繼續運算，直到較長的鏈結串列也遍歷完畢。
2. **邊界條件 (Final Carry)**：
   `while` 迴圈的條件包含了 `or carry`。這非常重要，例如 `5 + 5`，當 `l1` 和 `l2` 都遍歷完後，`carry` 為 1。迴圈會再執行一次，創建一個值為 1 的節點（結果為 `0 -> 1`），確保最高位的進位不會丟失。
3. **空輸入處理**：
   題目保證輸入非空，但即使輸入是 `[0]` 和 `[0]`，邏輯依然成立（輸出 `[0]`）。

## 複雜度分析
假設 `l1` 的長度為 $M$，`l2` 的長度為 $N$。

- **Time Complexity (時間複雜度): $O(\max(M, N))$**
  我們只需遍歷這兩個鏈結串列一次。迴圈執行的次數取決於較長的那個鏈結串列。

- **Space Complexity (空間複雜度): $O(\max(M, N))$**
  我們需要創建一個新的鏈結串列來儲存結果。結果鏈結串列的長度最多為 $\max(M, N) + 1$。
  *註：輔助空間 (Auxiliary Space) 為 $O(1)$，因為我們只用了固定數量的指標變數。*

## 常見誤區
1. **忘記處理最後的 Carry**：
   面試中最常見的錯誤是迴圈條件只寫 `while l1 or l2:`。這會導致最後一位進位遺失（如 `9+1` 變成 `0` 而不是 `0->1`）。
2. **程式碼冗餘**：
   避免寫三個分開的 `while` 迴圈（處理重疊、處理 l1 剩餘、處理 l2 剩餘）。使用 `if-else` 或一個統一的 `while` 邏輯能展現更好的代碼美感。
3. **指標丟失**：
   忘記執行 `cur = cur.next`，導致後續節點一直覆蓋在同一個位置。

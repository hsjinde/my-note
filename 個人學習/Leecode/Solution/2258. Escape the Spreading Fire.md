我在起點最多可以等多久，然後一路衝去安全屋，確保我比火早一步到？

## Solution 筆記
### 0. 參數定義

- a.網格相關
    - **row**：網格的行數（高）。
    - **col**：網格的列數（寬）。

- b.BFS隊列
    - **fire**：火焰蔓延的 BFS 隊列，儲存起火點及蔓延座標。
    - **path_queue**：主角逃脫路徑的 BFS 隊列，儲存移動座標與時間。

- c.輔助資料結構
    - **fireTime**：字典，記錄每個格子被火燒到的最早時間。
    - **passed**：集合，記錄主角已經走過的格子，避免重複走訪。

- d.移動方向
    - **direction**：四個方向的座標偏移量（上下左右）。



### 1. 找出起火點
- 遍歷整個 grid，把所有 `grid[x][y] == 1` 的座標加入 fire_queue

### 2. 計算起火蔓延時間
- 使用 BFS（queue）模擬火勢蔓延
- fireTime[(x, y)] 記錄火燒到每個格子的最早時間

### 3. 檢查是否可以在指定時間內逃脫（canEscape 函數）
- 建立函數，方便重複檢查不同等待時間
- **步驟：**
    1. 檢查起始格子 (0, 0) 是否在等待 waitTime 時已被燒到，若是則直接失敗（return False）
    2. BFS 尋找逃脫路徑，起點 (0, 0) 時間為 waitTime
    3. 每次從 queue 取出座標 (x, y) 和目前時間 currentTime
        - 若抵達終點 (row-1, col-1)，代表成功（return True）
        - 若曾走過該格，跳過
        - 標記已走過
    4. 依照四個方向嘗試移動（檢查邊界和是否已拜訪過）
    5. 判斷下一步 (nx, ny) 是否會被火燒到：
        - 若會：
            - 若是終點，可以和火同時到（fireTime >= nextTime）→ return True
            - 若不是終點，需比火早到（fireTime > nextTime）→ 加入 queue
        - 若不會，直接加進 queue

### 4. 二分搜尋最大等待時間
- 用 canEscape 檢查每個可能的等待時間，找出最大安全等待時間
- 若一開始就無法逃脫，return -1
- 若最大等待時間超過 row*col，return 10^9

---

### 關鍵補充

- BFS 走法和 visited/passed 集合避免重複計算
- 火蔓延和逃脫路徑分開模擬
- 終點可以和火同時到，但不能晚於火到

回傳值說明：
-1：無法逃脫
10^9：可以無限等待
其他數字：最大等待時間

這個解法的時間複雜度是 O(mn × log(mn))，空間複雜度是 O(mn)。

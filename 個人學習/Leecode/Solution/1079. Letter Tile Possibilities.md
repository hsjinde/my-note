## 題目描述 (中文摘要)
給定一個字串 `tiles`（長度 ≤ 7，字母為大寫 A-Z），每個字母代表一張牌。牌可以任意選擇與排列，但每張牌最多使用它原本出現的次數。請問可以組成多少個「非空」不同字串。

例如：  
輸入: `"AAB"`  
可形成：`A, B, AA, AB, BA, AAB, ABA, BAA` 共 8 個。

---

## 思路總覽
使用「回溯 (Backtracking) + 計數 (Counter)」：
1. 用 Counter/Frequency 表記錄每個字母剩餘可用數量。
2. 在 DFS 中，嘗試選擇任一個還有剩餘的字母：
   - 每選一次字母，即形成了一個新的「非空序列」，計數 +1。
   - 繼續遞迴探索更長的延伸。
   - 回溯恢復數量。
3. 枚舉的是一棵「排列前綴樹」，每個節點（除根節點外）代表一個唯一序列。

特點：
- 不需要用集合儲存所有字串（省記憶體）。
- 透過 frequency 控制避免重複。

---

## 演算法詳解
假設 tiles = "AAB"  
頻率表：{A:2, B:1}

遞迴樹（節點 = 已形成的序列）：

(根)  
├─ A  
│  ├─ AA  
│  │   └─ AAB (不成立，因為 B 只有 1 張且已用? ← 實際順序會依減法)  
│  └─ AB  
└─ B  
   └─ BA  
      └─ BAA (依頻率可合法)

實際程式不存字串，只透過「每往下一層選一個字母」時 res += 1。

## 程式碼實作
================================================== 中文

- 建立一個字母頻率表（dictionary / Counter），記錄每個字母剩餘可使用次數。
- 使用 DFS（回溯）遍歷：對每個還有剩餘次數的字母，
    - 將該字母次數減 1
    - 計數 +1（代表新增了一個新序列）
    - 繼續遞迴擴展
    - 回溯（次數加回）
- 直到所有字母都無法再選（本層迴圈沒有進入）即回傳 0；所有計數在回溯累積。
- 回傳總計數即為所有非空不同字串的數量。
- 時間複雜度最壞 O(n!)；空間 O(n)（遞迴深度），未存所有序列。
================================================== English Version (Completed Explanation)

Algorithm Steps

- Build a frequency map (dictionary / Counter) of the letters in tiles.
- Run a depth-first search (DFS) with backtracking:
    - For each letter that still has a remaining count > 0:
        a. Decrement its count (choose it).
        b. Add 1 to the answer (this newly formed prefix itself is a valid non-empty sequence).
        c. Recurse to try extending this prefix further.
        d. Restore the count (backtrack).
- The DFS returns 0 when no further letter can be chosen (natural termination via the for-loop).
- Sum over all branches to get the total number of distinct non-empty sequences.
 
- Time complexity is O(T) where T is the number of generated sequences; worst case (all unique letters) is O(n!) (more precisely Σ P(n, L), 1 ≤ L ≤ n). Space complexity is O(n) for the recursion stack (no need to store all sequences).

==================================================

### 原始 DFS 回溯版本
```python
from collections import Counter

class Solution:
    def numTilePossibilities(self, tiles: str) -> int:
        freq = Counter(tiles)
        chars = list(freq.keys())

        def dfs() -> int:
            total = 0
            for ch in chars:
                if freq[ch] == 0:
                    continue
                freq[ch] -= 1
                total += 1          # 以目前這個新增字母結尾的序列
                total += dfs()       # 延伸更長的序列
                freq[ch] += 1        # 回溯
            return total

        return dfs()
```

### 加入記憶化 (Memoization) 的版本（可選）
當字母種類較少但重複度高時，許多「頻率狀態」會重複出現，可以記憶化減少遞迴次數。

```python
from collections import Counter
from functools import lru_cache

class Solution:
    def numTilePossibilities(self, tiles: str) -> int:
        freq = Counter(tiles)
        chars = list(freq.keys())

        @lru_cache(None)
        def dfs(state):
            # state 是一個 tuple，代表各字母剩餘數量
            total = 0
            for i, cnt in enumerate(state):
                if cnt == 0:
                    continue
                # 選用第 i 個字母
                new_state = list(state)
                new_state[i] -= 1
                new_state = tuple(new_state)
                total += 1          # 使用該字母形成的新序列
                total += dfs(new_state)
            return total

        init_state = tuple(freq[c] for c in chars)
        return dfs(init_state)
```

備註：由於 n ≤ 7，原始版本已足夠；記憶化是示範思路。

## 正確性說明
- 每次遞迴代表「在當前前綴後再選一個字母」。
- 選字母後立即 +1，表示所有以該新字母結束的目前序列（不需要再儲存整個字串）。
- 不會重複：因為某一組剩餘頻率狀態下，對應的後續排列組合是唯一的；我們只會沿著「選取順序」自然展開一次。
- 未遺漏：所有合法使用次數的排列皆對應到一條唯一路徑。

## 時間複雜度分析
設：
- 總長度 n
- 不同字母數 k
- 各字母頻率 c1, c2, ..., ck，且 Σ ci = n

枚舉的序列總數 T 為：
T = Σ_{L=1..n} Σ_{(x1+...+xk = L; 0 ≤ xi ≤ ci)}  L! / (x1! x2! ... xk!)

這就是所有可形成的非空「不同排列」總數；演算法做的工作量為 O(T)。

最壞情況（所有字母皆不同，ci = 1）：  
長度 L 的序列數 = P(n, L) = n! / (n - L)!  
T = Σ_{L=1..n} P(n, L) < e * n! → 仍可表為 O(n!)。

因此：
- 最壞時間複雜度：O(n!)
- 一般情況：依字母重複度而下降。

記憶化版本：
- 狀態數量為 Π_{i=1..k} (ci + 1)
- 每個狀態最多嘗試 k 分支
- 上界 O( k * Π (ci + 1) )，對 n ≤ 7 通常小於原始 DFS 的展開。

---

## 空間複雜度分析
- 遞迴深度 ≤ n → O(n)
- freq / chars 結構 O(k) ≤ O(n)
- 未儲存所有結果（只計數），故不是 O(n!)。
- 記憶化版本需 O(Π (ci + 1)) 儲存 cache（仍極小，因 n ≤ 7）。

結論：空間複雜度 O(n)。

---

## 與常見誤區
1. 誤以為需要儲存所有字串：不需要，只需計數。
2. 誤判空間複雜度為 O(n!)：本解僅使用遞迴堆疊與頻率表。
3. 用 set 去重會多出大量記憶體與時間開銷，效率較差。
4. 認為需要生成全排列再去重：這樣會做無謂工作。

---

## 可延伸優化 / 思考方向
- 記憶化：當 k 小且某些字母頻率高時更有效。
- 轉為純組合 + 多項式係數直接計算：仍需枚舉各 (x1..xk) 分配，實作較繁瑣。
- 若題目改成輸出所有序列：需額外暫存當前字串，且複雜度與輸出量同階。

---

## 小結
- DFS + 回溯 + 頻率控制是最直觀且高效的寫法。
- 時間最壞 O(n!)，空間 O(n)。
- 不需用 set；不會重複計數。
- 可視需求加入記憶化降低重複狀態計算。

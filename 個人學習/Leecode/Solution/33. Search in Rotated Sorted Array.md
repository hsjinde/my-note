## 題目描述 (中文摘要)
給定一個升冪排序但被旋轉過的整數陣列 `nums`（無重複元素），以及一個整數目標值 `target`。  
請回傳 `target` 在陣列中的索引，若不存在則回傳 `-1`。  

範例：  
```
輸入: nums = [4,5,6,7,0,1,2], target = 0  
輸出: 4
```
```
輸入: nums = [4,5,6,7,0,1,2], target = 3  
輸出: -1
```

---

## 思路總覽
- 使用 **二分搜尋 (Binary Search)**。
- 陣列被旋轉，導致中點左右兩半其中之一仍是排序好的。
- 根據排序的一側判斷 `target` 是否位於該區間。
- 透過邏輯條件決定移動 `l` 或 `r`，直到找到或範圍縮小為空。

---

## 演算法詳解
1. **初始化指標**  
   設 `l = 0`, `r = len(nums) - 1`。

2. **二分搜尋主迴圈**  
   條件：`while l <= r:`  
   計算中點 `m = (l + r) // 2`。

3. **比對與區間判斷**  
   - 若 `nums[m] == target` → 回傳 `m`。
   - 若左半段有序 (`nums[l] <= nums[m]`)：  
     - 若 `nums[l] <= target < nums[m]` → 目標在左側，`r = m - 1`。  
     - 否則在右側，`l = m + 1`。
   - 否則右半段有序：  
     - 若 `nums[m] < target <= nums[r]` → 目標在右側，`l = m + 1`。  
     - 否則在左側，`r = m - 1`。

4. **未找到**  
   迴圈結束仍未回傳 → `return -1`。

---

## 程式碼實作
```python
from typing import List

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        l, r = 0, len(nums) - 1

        while l <= r:
            m = (l + r) // 2

            if nums[m] == target:
                return m
            elif nums[l] <= nums[m]:
                if nums[l] <= target < nums[m]:
                    r = m - 1
                else:
                    l = m + 1
            else:
                if nums[m] < target <= nums[r]:
                    l = m + 1
                else:
                    r = m - 1
        return -1
```

---

## 正確性說明
- 每次迴圈至少排除一半的區間，保證時間效率。
- 條件 `nums[l] <= nums[m]` 能準確辨識哪一側有序。
- 排除邊界後不會漏判或重覆計算，搜尋範圍單調收斂。

---

## 時間複雜度分析
- 每次迴圈縮小一半搜尋區間 → O(log n)。

---

## 空間複雜度分析
- 僅使用常數變數 → O(1)。

---

## 常見誤區與補充
1. **誤判旋轉側**：  
   若只比較 `nums[m]` 與 `nums[r]`，需對稱修改條件，否則錯誤。
2. **忘記等號**：  
   條件如 `nums[l] <= target < nums[m]` 需包含等號確保邊界正確。
3. **忽略無旋轉情況**：  
   若陣列本身升冪未旋轉，演算法仍可正確處理。

---

## 延伸思考
- 若存在重複元素 (`nums` 可重複)，需特別處理 `nums[l] == nums[m] == nums[r]` 的情況，可逐步縮左右指標。
- 此題是「旋轉二分搜尋」的經典模板，可延伸應用於：
  - 找最小值 (`findMin`)
  - 判斷是否存在 (`searchII`)

---

## 小結
- 利用二分搜尋結合旋轉特性。
- 透過有序區段判斷搜尋方向。
- 時間 O(log n)，空間 O(1)，為最優解。
